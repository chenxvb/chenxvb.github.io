    • Digital Communications (AD)

      • Easy to regenerate the distorted signal
        • Transmitted signals get distorted/degraded over distance
        • Regenerative repeaters along the transmission path can detect a digital signal and retransmit a new, clean (noise free) signal
        • This is not possible with analog communication systems
      • Data compression, error detection and error correction is possible
        • Digital communication is rugged in the sense that it is more immune to channel noise and distortion
      • Flexible Hardware is more
        • Digital hardware implementation is flexible and permits the use of microprocessors, mini-processors,
        • digital switching and VLSI
        • Shorter design and production cycle
      • Low cost
        • The use of LSI and VLSI in the design of components and systems have resulted in lower cost
        • Cost decreases over time
      • Easier and more efficient to multiplex several digital signals
        • Digital multiplexing techniques – Time & Code Division Multiple Access - are easier to implement than analog techniques such as Frequency Division Multiple Access
      • Can combine different signal types – data, voice, text, etc.
        • Data communication in computers is digital while voice communication between people is analog
        • Data & voice are difficult to combine over the same medium in the analog domain.
        • Using digital techniques, it is possible to combine both formats for transmission over the same medium
      • Larger storage capability
      • Better privacy and security using encryption
      • Better overall performance

    • Cons of Digital Comm

      • Requires larger bandwidth
      • Requires reliable “synchronization”
      • High power consumption due to multiple stages
      • It has complex circuitry

  • Lecture 2-3 Channel Coding

    • 信道编码，介绍了纠错编码（线性分组码纠错）和 ARQ 请求重传的两种纠错（信道上只实现了检错）

    • Error Control 1: Forward Error Correction (FEC)

      • 纠错编码，单次发送就能纠正错误（理想情况），通过增加冗余来实现

      • Key:

        • Transmit enough redundant data to allow receiver to recover from errors all by itself
        • No sender retransmission required

      • Major categories of FEC

        • Linear block codes 线性分组码
          • Cyclic codes
          • Reed-Solomon codes
        • Convolutional codes
        • Turbo codes
        • LDPC

    • Error Control 2: utomatic Repeat Request (ARQ)

      • 报错重传，接收方发现问题会要求重传，实际用的比较多。比如 tcp 就用的这个

      • Receiver must ask for retransmission – Automatic Repeat Request (ARQ)

      • Retransmission is needed if: Frame is erroneous or Frame is lost

        • D sends (positive) acknowledgement ACK if received packets are correct
        • If D detects errors, D sends negative acknowledgement NAK – this requests retransmission

      • Block

        • 经典 ARQ 算法，通信网络中考虑过这些情况。一个个传最简单但是慢，快的话会乱序导致的性能损害可能不如之前的。具体情况具体讨论
        • {:height 252, :width 679}

    • FEC vs ARQ

      • ARQ is much simpler than FEC and need no redundancy

      • ARQ is sometimes not possible if

        • A reverse channel is not available or the delay with ARQ would be excessive 比如说卫星通信基本不可能 ARQ
        • The retransmission strategy is not conveniently implemented
        • The expected number of errors, without corrections, would require excessive retransmissions

    • Linear Block Code Decoding 线性分组码

      • 感觉没啥时间详细写，反正信息论也学过。大意就是 m 和 G 生成矩阵 来产生 codeword, 此外 G 的逆矩阵 H 用来纠错。如果传输错误发生，r 和 H 的矩阵相乘结果会落在特定的区间内，在部分区间可以纠错
      • Generator Matrix G
      • Parity check matrix H

    • Error Detection and Correcting Capability

      • 这些最重要的还是冗余，主要纠错检错能力还是看每个 codeword 空了多少，也就是最短汉明距离的大小
      • The error-correcting capability t of a code: the maximum number of guaranteed correctable errors per codeword 纠错的最短汉明距离

  • Lecture 6-7: Modulation

    • 以 AM 和 FM 为例子，主要讲了三种数字调试手段，幅度的 ASK, 频率的 FSK, 相位的 PSK, 其中 PSK 还可以组合用，拥有更大的带宽
    • modulation 应该是在这里：
    • AM bandwidth: AM signal is equal to twice the bandwidth of the source signal.

    • Digital Modualtion: Amplitude Shift Keying (ASK)

      • Advantages: simplicity of implementation
      • Disadvantages: very susceptible to noise interference because noise affects the amplitude

      • Binary Amplitude Shift Keying (BASK)

      • Detectors for ASK – Coherent Receiver 同步
      • Detectors for ASK – Noncoherent Receiver 不需要同步

    • Frequency Shift Keying (FSK)

      • Advantages:
        • less susceptible to errors than ASK. 可能是因为噪声在频域平稳分布（相较于 ASK）
        • Receiver looks for specific frequency changes over a number of intervals so amplitude(noise) spikes can be ignored.
      • Disadvantages: uses twice the bandwidth as compared to ASK
      • Applications: early telephone modems, in high frequency radio transmission etc.
      • Until recently, FSK has been the most widely used form of digital modulation; Why?
        • Simple both to generate and detect
        • Insensitive to amplitude fluctuations in the channel

    • Phase Shift Keying (PSK)

      • PSK 现在都是更为主流的调制方法了
      • Advantages:
        • More power efficient
        • High data rate
        • Less prone to errors compared to ASK
      • Disadvantages:
        • It is complex to implement
        • Sensitive to phase difference
        • It has lower bandwidth efficiency compared to ASK
      • Applications: optical communications, wireless LAN, cellular, IoT etc.

      • Differential Phase Shift Keying (DPSK)

        • Applications of QAM

          • High-speed internet

          • QAM is more complex to implement
          • It consumes more power

    • 感觉讲了实际应用中的分组方式，全双工半双工什么的，题目会考下面的那些

    • Duplexing

      • FDD: Frequency division duplexing
      • TDD: Time division duplexing

    • Multiples Access Schemes

      • FDMA : Frequency division Multiple Access
      • TDMA : Time division Multiple Access
      • SSMA/CDMA : Spread spectrum Multiple Access

    • Frequency Division Duplexing (FDD)

      • {:height 106, :width 486}
      • 从频域分隔

    • Time Division Duplexing (TDD)

      • {:height 108, :width 481}
      • 从时域分割
      • Communication is not full-duplex

    • Trade-offs Between FDD and TDD

      • FDD
        • Provides individual radio frequencies to each user hence, transceiver should work on two frequency bands
        • Duplexer needed, more expensive
        • No additional time delays and latency (channels are always open)
      • TDD
        • Single frequency hence simple transceiver
        • Duplexer not needed, a switch can do the job
        • There is time latency, communication is not full-duplex

    • Frequency Division Multiple Access (FDMA)

      • It is expensive because a duplexer is needed in base station and subscriber units
      • FDMA requires RF filtering to minimize adjacent channel interference
      • Complexity of FDMA mobile systems is lower compared to TDMA
      • FDMA requires fewer bits for synchronization and framing

    • Time Division Multiple Access (TDMA)

      • TDMA uses time slots for the whole bandwidth
      • Users are separated in time and one user is allocated per slot
      • Transmission occurs in bursts
      • Transmission is non-continuous due to the time slot allocation
      • Total bandwidth is utilized
      • Bandwidth can be supplied on demand
      • Low power consumption
      • Duplexers are not required

  • Lecture 9 Line Coding

    • 这章东西真的多，但是主要是要记忆的。讲了根据不同标准进行的数字调制。这里大概是在编码和解调层之间。分析了不同调制方法下的 snr 和 功率的不同
    • 我猜应该在这里：

    • Line Coding

      • line coding 是一个总称
      • Line coding is the mapping of binary information sequence into the digital signal that enters the channel
      • It converts binary data (sequence of bits) into a digital signal
      • Digital signals depend on individual information bits, i.e., transmission is done bit-by-bit, as opposed to block coding

    • Properties of Line Coding

      • Self-synchronisation
        • Ability to recover timing from the signal itself
        • Long series of 1s and 0s can cause problems
      • Low probability of bit error
        • Receiver can recover binary data when the input data signal is corrupted
      • Suitable spectrum for the channel
        • Minimize transmission bandwidth
        • DC component should be avoided if channel has DC blocking capacitance
    • Line Coding – Unipolar Code
      • 最平常的编码，amplitude 为零信号为零，amp 高于一定量为 1。但是信号含有直流分量
      • Uses one non-zero and one zero voltage levels to represent data
      • For example, 0 for zero level and 1 for non-zero level
      • It is very simple to implement
      • However, it suffers from DC component and lack of synchronization in long series of 1’s and 0’s
    • Line Coding – Polar Code
      • ploar，带极性的编码，分成很多种。这个就没有直流分量
      • Uses two non-zero voltage levels to represent data levels i.e., positive and negative
      • This solves the DC problem of unipolar code
      • There are 4 main types of polar codes
        • Non return to zero (NRZ)
        • Return to zero (RZ)
        • Manchester
        • Differential Manchester
      • NRZ:
        • Polar NRZ-level: signal level represents particular bit, e.g., positive for 1 and negative for 0. This lacks synchronisation for long series of 1’s and 0’s
      • NRZ-inverted:
        • 根据变化来存储信息，如果信号没有变就是 0，变了就是 1
        • inversion of voltage level denotes bit 1, while 0 has no change
      • Polar return to zero (RZ):
        • 另一个非常朴素的方法来避免直流分量，但是浪费了编码空间。如果我都能准确控制两个起伏来发送信息，那么我为啥不拿来整两个 bit 呢
        • {:height 165, :width 391}
        • signal level represents particular bit, e.g., positive for 1 and negative for 0
        • Signal must return to zero half-way through the bit interval
        • This results in perfect synchronisation
        • However, 2 signal changes used to encode each bit pulse rate
        • This means it is twice the rate of NRZ coding, i.e., more bandwidth is required, regardless of bit sequence
      • Manchester code:
        • 0 变 1 为 0，1 变 0 为 0。但全是 1 或者 0 的话很浪费带宽（参考上面）
        • {:height 109, :width 604}
        • both synchronisation and bit representation are achieved using inversion at the middle of each bit interval
        • E.g., positive to negative = 1; negative to positive = 0
        • Works well for alternating sequences of bits, e.g., 101010, but wastes bandwidth for long runs of 1’s and 0’s
        • It is used by Ethernet (IEEE 802.3)
      • Differential Manchester code: inversion in the middle of bit interval is used for synchronization
        • 和上面反过来。但这次 1 或者 0 的话节约，变化频繁很浪费
        • {:height 127, :width 598}
        • The presence or absence of additional transition at the beginning of next bit interval identifies the bit
        • Works well for long runs of 1s, but wastes bandwidth for long runs of 0’s

    • Line Coding – Bipolar Code

      • 典
      • {:height 142, :width 538}
      • It uses 2 non-zero and zero voltage levels for representation of two data levels
      • Zero is denoted by 0 V level, and 1 is denoted by positive and negative voltage levels
      • If the first 1 is represented by positive value, the next 1 will be represented by negative voltage, next will be positive, etc.,
      • It uses less bandwidth compared to Manchester, but loss of synchronization is possible for long runs of 0s

    • Three basic equalization methods

      • Linear equalization (LE):
        • Performance is not very good when the frequency response of the frequency selective channel contains deep fades.
      • Zero-forcing algorithm
        • aims to eliminate the intersymbol interference (ISI) at decision time instants (i.e. at the center of the bit/symbol interval).
      • Least-mean-square (LMS) algorithm
        • will be investigated in greater detail in this presentation.
      • Recursive least-squares (RLS) algorithm
        • offers faster convergence, but is computationally more complex than LMS (since matrix inversion is required).
      • Decision feedback equalization (DFE):
        • {:height 218, :width 617}
        • Performance better than LE, due to ISI cancellation of tails of previously received symbols.
      • Maximum Likelihood Sequence Estimation using the Viterbi Algorithm (MLSE-VA):
        • Best performance. Operation of the Viterbi algorithm can be visualized by means of a trellis diagram with $$m^{K-1}$$ states, where m is the symbol alphabet size and K is the length of the overall channel impulse response (in samples).

    • 计算题经常用到下面的公式，这边写一下对应的意义
      • B 为 bandwidth，既为本次 OFDM 的频带宽度，它的倒数是本次的采样频率 $$T_s$$
      • 对于 $$\Delta f $$ ，它是 subcarrier spacing，也就是每个载波的间隔。很显然，如果给定了载波的个数，那么这个的大小就等于 整个频带宽度除以数量 $$\Delta f= B / N$$ ，它的倒数是整个周期 T。而且这个周期还是一个 symbol 的周期，详见可以想想 QPSK 是如何调制的

      • symbol rate 的公式为 (前面的)： $$Symbol\ Rate = Code Rate * 1 / T$$
      • 因为同一时间有多个 symbol 同时发送，所以 OFDM 系统的 SR 为 N * 上面的 symbol rate
